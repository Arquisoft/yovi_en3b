== Runtime View

=== User Login

This diagram the action of the user login

[plantuml,"Login",png]
----
@startuml
actor User
participant WebApp
participant UsersService
database Database

User -> WebApp: Enters credentials (username/password)
WebApp -> UsersService: Auth Request
UsersService -> Database: Search for user & validate password hash
Database --> UsersService: Return user data (or error)

alt valid credentials
    UsersService --> WebApp: Login Success
    WebApp --> User: Redirect to Game Dashboard
else invalid credentials
    UsersService --> WebApp: Login Failed (401 Unauthorized)
    WebApp --> User: Show "Invalid username or password"
end
@enduml
----

=== User plays the game

This diagram shows the execution of a game

[plantuml,"Game",png]
----
@startuml
actor User
participant WebApp
participant GameYEngine
participant UsersService
database Database

User -> WebApp: Selects board size, difficulty
User -> WebApp: Selects 'Play'
WebApp -> GameYEngine: Requests the board
GameYEngine --> WebApp: Returns the board

loop
    User -> WebApp: Plays their turn
    WebApp -> GameYEngine: Sends the board state & Checks if there is a winner
    GameYEngine -> WebApp: Returns whether there is a winner
    break There is a winner!
        WebApp --> User: Shows result of the game
    end
    WebApp -> GameYEngine: Sends the board state & Asks the bot to play
    GameYEngine --> WebApp: Returns the play of the bot 
    WebApp --> User: Shows the play of the bot
end

WebApp -> UsersService: Asks to save the data of the game
UsersService -> Database: Asks to save the data of the game
Database --> UsersService: Confirms the data was saved
UsersService --> WebApp: Confirms the data was saved
@enduml
----

This diagram represents the user asking for a hint

@startuml

User -> WebApp: Asks for a hint
WebApp -> GameYEngine: Sends the board state & Asks for the move

@enduml
----
